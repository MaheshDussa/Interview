# 12 — Build, Deploy & Production

> **One-liner**: `ng build` produces a static `dist/` folder. Serve from **Azure Static Web Apps / App Service / Container Apps / CDN-fronted Blob Storage**. Add SSR only when SEO or first paint require it.

---

## 1. Production Build

```powershell
ng build                                  # production by default (Angular 17+)
ng build --configuration=development
ng build --base-href=/myapp/              # if hosted under subpath
ng build --stats-json                     # for bundle analysis
```

Output: `dist/<project>/browser/` (and `dist/<project>/server/` if SSR).

Key flags applied in production config (from `angular.json`):
- `optimization: true` (minify, tree-shake, prefix CSS)
- `outputHashing: 'all'` (cache busting via file-name hash)
- `sourceMap: false` (or `true` + upload to monitoring)
- `extractLicenses: true`
- `budgets` (initial/component size limits)

## 2. Bundle Budgets

```jsonc
"budgets": [
  { "type": "initial",     "maximumWarning": "500kb", "maximumError": "1mb" },
  { "type": "anyComponentStyle", "maximumWarning": "4kb", "maximumError": "8kb" }
]
```

CI fails when bundles bloat — a senior must keep these tight.

Analyze:
```powershell
npm i -D source-map-explorer
ng build --stats-json
npx source-map-explorer dist/my-app/browser/*.js
```

## 3. Environment Configuration

Two strategies — **build-time** vs **runtime**.

### Build-time (file replacement)
```jsonc
"configurations": {
  "production": {
    "fileReplacements": [
      { "replace": "src/environments/environment.ts",
        "with":    "src/environments/environment.prod.ts" }
    ]
  }
}
```
One build per environment.

### Runtime config (recommended for containers)
Ship one image, mount/serve a JSON config:
```ts
// app.config.ts
provideAppInitializer(() => {
  const cfg = inject(ConfigService);
  return cfg.load();                       // fetch /assets/config.json
});
```
Container entrypoint can substitute `assets/config.json` from env vars before nginx starts.

> Senior interview line: "Runtime config beats build-time for Docker images — one image promotes through dev/test/prod by mounting different config."

## 4. Deploy Target Comparison

| Target | When | Pros | Cons |
|---|---|---|---|
| **Azure Static Web Apps** | Pure SPA, GitHub/ADO PR previews | Free tier, integrated auth, edge CDN, free SSL | API must be Azure Functions / external |
| **Azure App Service (static)** | Mixed with backend, easy | Slots, easy custom domain | Pay even for static |
| **Azure Blob Storage + Front Door / CDN** | Cheap global SPA | Pennies, fast | Manual SPA routing rewrite rules |
| **Azure Container Apps** | SSR Angular Universal | KEDA scale, revisions | More infra |
| **AKS** | Multi-app / enterprise | Full K8s | Operational cost |
| **Nginx / IIS on VM** | Legacy | Total control | You patch the OS |

## 5. Azure Static Web Apps (recommended for SPAs)

```powershell
az staticwebapp create -n my-app -g rg1 -l "Central US" \
  --source https://github.com/me/repo --branch main \
  --app-location "/" --output-location "dist/my-app/browser" \
  --login-with-github
```

`staticwebapp.config.json` (SPA fallback, headers, auth):
```jsonc
{
  "navigationFallback": { "rewrite": "/index.html", "exclude": ["/assets/*"] },
  "globalHeaders": { "Cache-Control": "no-cache" },
  "routes": [
    { "route": "/admin/*", "allowedRoles": ["admin"] }
  ],
  "responseOverrides": { "404": { "rewrite": "/index.html" } }
}
```

GitHub Actions workflow is auto-generated by the create command.

## 6. App Service (static or container)

Deploy build folder as a zip:
```powershell
Compress-Archive dist/my-app/browser/* deploy.zip
az webapp deploy -g rg1 -n my-app --src-path deploy.zip --type zip
```

Or as a container:
```powershell
az webapp create -g rg1 -p plan1 -n my-app \
  --deployment-container-image-name myacr.azurecr.io/my-app:v1
az webapp config appsettings set -g rg1 -n my-app --settings WEBSITES_PORT=80
```

**SPA URL fallback** — App Service needs a `web.config` with rewrite rules so deep links work.

## 7. Blob Storage + Front Door (cheap & fast)

```powershell
az storage account create -n myappsa -g rg1 -l eastus --kind StorageV2 --sku Standard_LRS
az storage blob service-properties update --account-name myappsa --static-website \
  --index-document index.html --404-document index.html
az storage blob upload-batch -d '$web' -s dist/my-app/browser --account-name myappsa --overwrite
```

Add Azure Front Door / CDN for global cache + custom domain + WAF.

Cache strategy: hashed files (`main.abc123.js`) → `Cache-Control: max-age=31536000, immutable`; `index.html` → `Cache-Control: no-cache`.

## 8. Dockerfile (Nginx, multi-stage)

```dockerfile
# build
FROM node:20-alpine AS build
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build -- --configuration=production

# runtime
FROM nginx:1.27-alpine
COPY --from=build /app/dist/my-app/browser/ /usr/share/nginx/html/
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
HEALTHCHECK CMD wget -qO- http://localhost/ || exit 1
```

`nginx.conf` with SPA fallback + cache headers:
```nginx
server {
  listen 80;
  root /usr/share/nginx/html;

  location / {
    try_files $uri $uri/ /index.html;
  }

  location ~* \.(?:js|css|woff2?|png|jpg|svg)$ {
    expires 1y;
    add_header Cache-Control "public, immutable";
  }

  location = /index.html {
    add_header Cache-Control "no-cache";
  }

  gzip on;
  gzip_types text/plain application/javascript text/css application/json image/svg+xml;
}
```

Build & push:
```powershell
docker build -t myacr.azurecr.io/my-app:v1 .
docker push myacr.azurecr.io/my-app:v1
```

## 9. SSR / Angular Universal (Production)

Add SSR:
```powershell
ng add @angular/ssr
```

This adds:
- `server.ts` (Express server)
- `main.server.ts`
- `app.config.server.ts`
- Build target `serve-ssr` and `build` produces both `browser/` and `server/`

Run prod SSR:
```powershell
node dist/my-app/server/server.mjs
```

Dockerfile for SSR:
```dockerfile
FROM node:20-alpine AS build
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM node:20-alpine
WORKDIR /app
COPY --from=build /app/dist /app/dist
COPY --from=build /app/node_modules /app/node_modules
ENV NODE_ENV=production PORT=4000
EXPOSE 4000
CMD ["node", "dist/my-app/server/server.mjs"]
```

Deploy to Azure Container Apps:
```powershell
az containerapp create -g rg1 -n my-app-ssr \
  --environment env1 \
  --image myacr.azurecr.io/my-app-ssr:v1 \
  --target-port 4000 --ingress external \
  --min-replicas 1 --max-replicas 10
```

## 10. Pre-rendering (SSG)

For mostly-static sites with a few dynamic pages:
```powershell
ng build --prerender              # since v17
```

Or specify routes in `app.routes.ts` with `RenderMode.Prerender` (Angular 18+).

```ts
// app.routes.server.ts
import { RenderMode, ServerRoute } from '@angular/ssr';
export const serverRoutes: ServerRoute[] = [
  { path: '',           renderMode: RenderMode.Prerender },
  { path: 'blog/:slug', renderMode: RenderMode.Prerender, getPrerenderParams: () => [{slug:'a'},{slug:'b'}] },
  { path: 'user/:id',   renderMode: RenderMode.Server },
  { path: '**',         renderMode: RenderMode.Client }
];
```

## 11. CI/CD with GitHub Actions

```yaml
name: deploy
on: { push: { branches: [main] } }
jobs:
  build-deploy:
    runs-on: ubuntu-latest
    permissions: { id-token: write, contents: read }
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: 20, cache: 'npm' }
      - run: npm ci
      - run: npm run lint
      - run: npm test -- --watch=false --browsers=ChromeHeadless --code-coverage
      - run: npm run build -- --configuration=production

      # Static Web Apps
      - uses: Azure/static-web-apps-deploy@v1
        with:
          azure_static_web_apps_api_token: ${{ secrets.SWA_TOKEN }}
          app_location: "/"
          output_location: "dist/my-app/browser"
          repo_token: ${{ secrets.GITHUB_TOKEN }}

      # OR Container Apps via OIDC
      - uses: azure/login@v2
        with:
          client-id: ${{ secrets.AZURE_CLIENT_ID }}
          tenant-id: ${{ secrets.AZURE_TENANT_ID }}
          subscription-id: ${{ secrets.AZURE_SUB }}
      - run: az acr build -r myacr -t my-app:${{ github.sha }} .
      - run: az containerapp update -g rg1 -n my-app --image myacr.azurecr.io/my-app:${{ github.sha }}
```

OIDC = no long-lived secrets in GitHub.

## 12. Performance Checklist Before Going Live

- [ ] `ng build --configuration=production` (default)
- [ ] OnPush + signals everywhere
- [ ] Lazy routes + preloading
- [ ] `NgOptimizedImage` for images
- [ ] Bundle budgets enforced
- [ ] HTTP caching: hashed assets immutable, `index.html` no-cache
- [ ] CDN in front (Front Door / Akamai / Cloudflare)
- [ ] Gzip/Brotli enabled on server
- [ ] HSTS, CSP, X-Content-Type-Options, Referrer-Policy headers
- [ ] Lighthouse > 90 on mobile

`staticwebapp.config.json` / `nginx.conf` for security headers:
```nginx
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
add_header X-Content-Type-Options nosniff;
add_header X-Frame-Options DENY;
add_header Referrer-Policy strict-origin-when-cross-origin;
add_header Content-Security-Policy "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; img-src 'self' data: https:;";
```

## 14. Internationalization (i18n)

Two main options:

| | Built-in `@angular/localize` | Transloco / ngx-translate |
|---|---|---|
| When | Compile-time, multiple builds per locale | Runtime, single build |
| Trade-off | Smaller bundles, faster | Easier dynamic switching |

```html
<h1 i18n="@@welcome">Welcome</h1>
```
```powershell
ng add @angular/localize
ng extract-i18n --output-path src/locale
ng build --localize
```

## 15. Versioning, Rollback, Zero Downtime

- **Static Web Apps / App Service slots** — swap to deploy, swap back to roll back.
- **Container Apps revisions** — keep N revisions, route 90/10 for canary.
- **Blob + Front Door** — upload to `v123/` folder, flip Front Door origin path.

> Always serve `index.html` with `Cache-Control: no-cache`. New JS files have new hashes so users always boot the latest version.

## 16. Common Deploy Pitfalls

| Pitfall | Fix |
|---|---|
| Deep link 404s (`/users/1` reload → 404) | SPA fallback to `index.html` on hosting |
| Stale app forever after deploy | `index.html` must NOT be cached |
| Hard-coded API URL per env | Use runtime config JSON |
| Different builds per env => regression risk | One build, runtime config |
| Mixed-content errors | Force HTTPS; relative URLs or env-based base URL |
| OAuth redirect URI mismatch | Add prod URL in Entra app registration |
| Missing source maps in production debug | Upload sourcemaps to monitoring, don't ship publicly |
| CSP blocks inline scripts | Use Angular hash-based or `nonce` per request |
| Bundle blew the budget | Check source-map-explorer, lazy load, drop heavy lib |
| `WEBSITES_PORT` not set for container | Set to match `EXPOSE` |

## 17. Senior Interview Q&A

**Q1. How does Angular handle production builds differently?**
Enables AOT compilation (ahead-of-time), tree-shaking, dead-code elim, minification, file hashing for cache busting, ES2022 target, and strict template checks.

**Q2. AOT vs JIT — when does each apply?**
AOT compiles templates at build time → smaller bundle, faster startup, errors caught early. JIT compiles in the browser (only used during `ng test` historically). Modern Angular is AOT everywhere.

**Q3. What's `ng build --prerender`?**
Generates static HTML for the routes you specify at build time. Best for SEO + fast first paint without running a Node server in prod.

**Q4. When do you need SSR?**
SEO (search engines / OG meta), social previews, very fast LCP on slow networks, or content gates that need server-side decisions.

**Q5. How to host an Angular SPA on Azure?**
**Static Web Apps** is the default — free, integrated PR previews, custom domain, free SSL. For SSR, use **Container Apps** or **App Service** with a Node container.

**Q6. Hash-based vs path-based routing?**
Default is path-based (`/users/1`) — needs SPA fallback on the host. Hash-based (`#/users/1`) avoids server config but is uglier. Production uses path-based.

**Q7. How would you cut Time to Interactive?**
Lazy routes, preloading on idle, smaller bundles (drop heavy libs), `NgOptimizedImage`, CDN, HTTP/2, deferred third-party scripts, SSR if SEO/LCP demand it.

**Q8. CI/CD strategy?**
Trunk-based, PR builds with preview URLs (SWA does this free), lint+test+e2e in CI, OIDC to Azure, immutable image tagged with git SHA, atomic deploy via slot swap or revision flip.

**Q9. How to do feature flags safely?**
Runtime config + a `FeatureFlagService`, gate by user/role, default OFF. Optionally lazy-load feature modules only when the flag is on (via `canMatch`).

**Q10. How to roll back a bad deploy?**
Static Web Apps: redeploy previous artifact. App Service: swap slot back. Container Apps: route traffic to previous revision (`--revision-mode multiple`). Blob+FrontDoor: change origin path.
